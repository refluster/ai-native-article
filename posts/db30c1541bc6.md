# Claude CodeとCoworkの「Mac遠隔操作」研究プレビュー——コネクター不在時は画面を見て操作、OpenClawとは権限設計が異なる

## Executive Summary

MacRumorsによると、Anthropicは「Claude Code」と「Cowork」において、ユーザー不在時にClaudeがMacを遠隔操作できる機能を追加した。アプリ向けの直接コネクターがない場合、Claudeは人間のように画面を見て「point, click, and navigate」するフォールバックを使う。

この機能は、iPhoneからタスクを投げる「Dispatch」と組み合わせる前提で示されており、同時にAnthropic自身が「Computer use is still early」と明言している。OpenClawとの比較では、機能の近さよりも、安全な構成難易度と権限付与の方式の違いが前面に出ている。

## Claude Code+Coworkの新機能: Macを「人間のように」操作するフォールバック

Anthropicは、Claudeが必要なツールに直接アクセスできない場合でも、Macの画面操作を通じて作業を完了する仕組みを導入した。これはAPI連携の拡張ではなく、スクリーン上のUIを辿る代替経路として説明されている。

- **エビデンス**
- MacRumorsは「Anthropic's Claude AI Can Now Use Your Mac While You're Away」と報じている。
- 記事では「the company's Claude Code and Cowork tools can now remotely control your Mac on your behalf」とある。
- 直接コネクターがない例として「Slack or Google Calendar」が挙げられている。
- その場合、Claudeは「controlling the computer like a human, using the screen to navigate」と説明される。
- Claudeブログからの引用として、以下が示されている。
- 「In Claude Cowork and Claude Code, you can now enable Claude to use your computer to complete tasks.」
- 「When Claude doesn't have access to the tools it needs, it will point, click, and navigate what's on your screen to perform the task itself.」
- 「It can open files, use the browser, and run dev tools automatically – with no setup required.」
- **結論**
- 中核は「コネクターの追加」ではなく、「コネクターがなくても画面経由で作業を継続する」ことにある。
- 「with no setup required」という表現から、利用開始の障壁は低く設計されている一方、操作対象はOS上の実アプリとUIに及ぶ。
- Claudeの役割は、テキスト応答やコード生成から、Mac上の実行主体へ拡張されている。
## Dispatchとの組み合わせ: iPhoneから依頼し、Macで処理を終える運用

この機能は単独のデモではなく、先週公開された「Dispatch」と接続されている。操作の起点をiPhoneに置き、処理の終点をデスクトップに置く使い方が明示されている。

- **エビデンス**
- 記事では「The capability pairs with Dispatch (released last week)」とある。
- Dispatchは「lets you assign Claude tasks from your iPhone and return to finished work on your desktop」と説明されている。
- 埋め込みYouTube動画のデモでは、ユーザーがClaudeに「export a pitch deck as a PDF and attach it to a meeting invite」を依頼し、「while the user is away from their Mac」とされている。
- **結論**
- この機能の利用文脈は、Macの前で逐次指示する対話型操作ではなく、離席中の非同期実行である。
- iPhoneが指示端末、Macが作業端末という役割分担が前提化されている。
- 「pitch deck」「PDF」「meeting invite」という一連の処理例から、単一アプリ内の操作ではなく、ファイル処理とスケジューリングをまたぐタスク自動化が想定されている。
## OpenClawとの違い: 機能の類似よりも、安全性と権限制御の差

MacRumorsはこの機能を「Anthropic's version of OpenClaw」と位置づけつつ、両者の差分を安全な導入性に置いている。OpenClawは強力だが安全に構成するのが難しく、Claude側はより制限された設計だと整理されている。

- **エビデンス**
- 記事は「The new feature is essentially Anthropic's version of OpenClaw」と述べる。
- OpenClawについては、以下の説明がある。
- 「the open-source AI agent that went viral earlier this year」
- 「runs locally」
- 「connects to messaging apps like WhatsApp and Telegram」
- 「using a plugin-based "skills" system to execute tasks ranging from file management to browser automation」
- 「It's powerful, but notoriously tricky to configure safely.」
- Claude側については、以下の説明がある。
- 「Claude's version is more locked down.」
- 「The system uses a permission-first approach, and requests user access before touching a new app.」
- 「It's currently also Mac-only, whereas OpenClaw supports macOS, Windows, and Linux.」
- **結論**
- 両者は「コンピューター利用型エージェント」という点では近いが、導入思想は異なる。
- OpenClawはローカル実行・プラグイン拡張・複数OS対応で柔軟性が高い一方、「configure safely」が難所として示されている。
- Claudeの実装は「permission-first approach」と「Mac-only」によって、対象範囲と権限付与を絞り込む方向にある。
- したがって比較軸は、単純な機能量ではなく、可搬性と安全な既定値のどちらを優先するかにある。
## Anthropic自身の留保と提供条件: 「Computer use is still early」

Anthropicはこの機能を完成済みの自動化基盤としてではなく、誤作動と脅威進化を前提にした初期段階の機能として提示している。提供範囲も全ユーザー向けではなく、限定的である。

- **エビデンス**
- Anthropicは「Computer use is still early compared to Claude's ability to code or interact with text」と述べている。
- 続けて「Claude can make mistakes, and while we continue to improve our safeguards, threats are constantly evolving.」としている。
- 利用上の推奨として「We recommend starting with the apps you trust and not working with sensitive data.」とある。
- 提供条件は「now available in research preview for Claude Pro and Max subscribers」。
- **結論**
- ベンダー自身が、コード生成やテキスト対話と同等の成熟度には達していないと位置づけている。
- 想定されるリスクは、単なる操作ミスだけでなく、「threats are constantly evolving」と表現される継続的な安全保障上の問題を含む。
- 「apps you trust」「not working with sensitive data」という条件付きの推奨は、利用対象を限定した試験運用が前提であることを示す。
- 「research preview」という提供形態も、機能の一般化より検証段階に重心があることを裏づける。
## 今月の連続アップデートの文脈: ビジュアル対応とメモリ移行の延長線上にあるMac操作

Mac遠隔操作は単発の追加ではなく、今月のClaude拡張の一部として報じられている。視覚情報の扱いと他社AIからの文脈移行に続き、実環境での操作能力が加わった構図である。

- **エビデンス**
- 記事は「Earlier this month, Claude was updated with support for inline visual content that aims to help provide clearer answers.」と述べる。
- さらに「Anthropic also rolled out a memory import tool that allows users to import conversations and memories from other AI providers into Claude, so new users don't need to start from scratch when they switch.」とある。
- **結論**
- 今月の更新は、回答表現の拡張、履歴・記憶の移行、そしてMac上での実行能力という3方向にまたがっている。
- 「inline visual content」は画面や視覚情報を扱う文脈と接続しやすく、「memory import tool」は他社サービスからClaudeへの移行障壁を下げる位置づけになっている。
- その上で今回のMac操作機能は、Claudeを単なるチャット相手ではなく、既存のデスクトップ作業フローに入り込むエージェントとして押し出す更新と読める。
