# Claude Code「294 release notes」が示す、週次ではなく日次で回るAIネイティブ開発サイクル

## Executive Summary

ReleasebotのAnthropicアーカイブでは、最終更新日「May 1, 2026」時点で「Claude Code 294 release notes」「All Anthropic Release Notes (515)」と記録されている。2026年4月末から5月初頭にかけても、`2.1.119`、`2.1.120`、`2.1.121`、`2.1.122`、`2.1.123`、`2.1.126`が短期間に連続して公開されており、Claude Codeの開発は大型機能追加と細かな不具合修正を同一リズムで回している。

出典が示しているのは、AI製品の開発対象がモデル本体だけではなく、MCP、OAuth、OpenTelemetry、Bedrock、Vertex AI、Windows PowerShell、VSCode、Remote Control、hooks、plugin運用まで広がっている点である。つまり「AIアプリ」ではなく、外部接続・運用・権限・観測可能性を含む実運用スタック全体が、継続的に更新される前提で設計されている。

## 2.1.119〜2.1.126に集中する、連続リリースの実像

2026年4月下旬から5月1日にかけて、Claude Codeは高頻度でバージョンを積み上げている。単一の目玉機能を大きく出すというより、複数の機能面・運用面・安定性面を同時並行で更新している構図が読み取れる。

- エビデンス
- `2.1.119` は「First seen by Releasebot: Apr 24, 2026」
- `2.1.120` は「First seen by Releasebot: Apr 28, 2026」
- `2.1.121` は「First seen by Releasebot: Apr 28, 2026」
- `2.1.122` は「First seen by Releasebot: Apr 29, 2026」
- `2.1.123` は「First seen by Releasebot: Apr 29, 2026」
- `2.1.126` は「First seen by Releasebot: May 1, 2026」
- Releasebot集計では「Claude Code 294 release notes」
- 同期間の週次ダイジェストとして「Week 16」「Week 17」も掲載されている
- `Week 16` には「Opus 4.7 defaulting」「xhigh effort」「cloud routines」「parallel code review」「usage tracking」「native CLI binaries」
- `Week 17` には「ultrareview」「session recaps」「custom themes」「a redesigned web experience with a new sessions sidebar」
- 結論
- リリース単位は「四半期ごとの大型更新」ではなく、日次に近い粒度で刻まれている
- しかも週次ダイジェストと個別パッチが併存しており、要約レイヤーと実装レイヤーが分離されている
- これは、利用者向けには週単位で意味づけしつつ、実際の開発・出荷はそれより細かいサイクルで回していることを示す
## MCP・hooks・pluginが中心にある「接続前提」の製品設計

更新内容の中核には、MCPサーバー、claude.ai connectors、hooks、plugin運用が繰り返し現れる。Claude Codeは単体CLIではなく、外部ツールや認証済み接続先を束ねる実行環境として進化している。

- エビデンス
- `2.1.121`:
- 「Added alwaysLoad option to MCP server config」
- 「MCP servers that hit a transient error during startup now auto-retry up to 3 times」
- 「Claude.ai connectors with the same upstream URL are now deduplicated」
- 「PostToolUse hooks can now replace tool output for all tools via hookSpecificOutput.updatedToolOutput」
- 「Added claude plugin prune」
- `2.1.122`:
- 「/mcp now shows claude.ai connectors hidden by a manually-added server with the same URL」
- 「Clarified the /mcp message shown when an MCP server is still unauthorized after the browser sign-in flow」
- 「Fixed ToolSearch missing MCP tools that connected after session start in nonblocking mode」
- `2.1.119`:
- 「Subagent and SDK MCP server reconfiguration now connects servers in parallel instead of serially」
- 「Plugins pinned by another plugin's version constraint now auto-update to the highest satisfying git tag」
- 「Hooks: PostToolUse and PostToolUseFailure hook inputs now include duration_ms」
- `2.1.126`:
- 「The /model picker now lists models from your gateway's /v1/models endpoint when ANTHROPIC_BASE_URL points at an Anthropic-compatible gateway」
- 「Fixed claude.ai MCP connectors being suppressed by manual servers stuck in needs-auth state」
- 「Fixed deferred tools (WebSearch, WebFetch, etc.) not being available to skills with context: fork and other subagents on their first turn」
- 結論
- 開発対象は「モデル呼び出しUI」ではなく、接続の発見、再接続、重複排除、認可状態、並列再構成まで含む
- MCPやpluginが頻繁に更新されていることは、AIエージェントの価値が推論そのものだけでなく、周辺ツール接続の信頼性で決まることを示している
- hooksの入力拡張やtool output置換は、Claude Codeが固定製品というより、利用者側で振る舞いを組み替える基盤として扱われていることを示す
## OAuth・権限・セキュリティ修正が、機能追加と同じ頻度で流れている

認証、権限、保護パス、managed settings、テレメトリ抑制といった運用上の制御面が、機能追加と同じ密度で更新されている。AIツールの開発速度は、許可モデルと認証導線の改善速度でもある。

- エビデンス
- `2.1.123`:
- 「Fixed OAuth authentication failing with a 401 retry loop when CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS=1 is set」
- `2.1.126`:
- 「--dangerously-skip-permissions now bypasses prompts for writes to .claude/ , .git/ , .vscode/ , shell config files, and other previously-protected paths」
- 「claude auth login now accepts the OAuth code pasted into the terminal when the browser callback can't reach localhost (WSL2, SSH, containers)」
- 「Security: Fixed allowManagedDomainsOnly / allowManagedReadPathsOnly being ignored when a higher-priority managed-settings source lacked a sandbox block」
- 「Fixed OAuth login failing with timeout on slow or proxied connections, in IPv6-only devcontainers, and when the browser callback can't reach localhost」
- 「Fixed a rare race where a concurrent credential write could clear a valid OAuth refresh token」
- 「Windows: clipboard writes no longer expose copied content in process command-line arguments visible to EDR/SIEM telemetry」
- `2.1.121`:
- 「SDK: mcp_authenticate now supports redirectUri for custom scheme completion and claude.ai connectors」
- 「Fixed Microsoft 365 MCP OAuth failing with duplicate or unsupported prompt parameter」
- 「Fixed managed settings approval prompt exiting the session even when accepted — now applies settings and continues」
- `2.1.120`:
- 「Fixed DISABLE_TELEMETRY / CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC not suppressing usage metrics telemetry for API and enterprise users」
- `2.1.119`:
- 「Security: blockedMarketplaces now correctly enforces hostPattern and pathPattern entries」
- 結論
- 認証導線の改善は補助的な修正ではなく、CLI・コンテナ・SSH・WSL2・プロキシ環境で使うための本体機能である
- `--dangerously-skip-permissions` の適用範囲拡大と安全ネット維持の併記は、速度と保護の両立を細かく調整していることを示す
- EDR/SIEMやmanaged settingsへの言及は、Claude Codeの想定利用環境が個人開発だけでなく、監査や統制のある組織環境にあることを示している
## OpenTelemetry・usage・Rate Limits APIに見る「観測可能性の先行実装」

出典には、OpenTelemetry属性の追加、usage可視化、Rate Limits API公開が並んでいる。AI製品の改善サイクルは、モデル精度だけでなく、利用状況・失敗理由・制限状態をどれだけ観測できるかに依存している。

- `2.1.122`:

- 「OpenTelemetry: added claude_code.at_mention log event for @ -mention resolution」
- `2.1.121`:
- 「OpenTelemetry: added stop_reason , gen_ai.response.finish_reasons , and user_system_prompt (gated behind OTEL_LOG_USER_PROMPTS ) to LLM request spans」
- `2.1.119`:
- 「OpenTelemetry: tool_result and tool_decision events now include tool_use_id ; tool_result also includes tool_input_size_bytes」

- `2.1.126`:
- 「claude_code.skill_activated OpenTelemetry event now fires for user-typed slash commands and carries a new invocation_trigger attribute」
- 「better telemetry」と要約されている
- `Week 16`:

- 結論
- 何が起きたかを記録するだけでなく、`tool_use_id`、`tool_input_size_bytes`、`invocation_trigger` のように原因追跡に使える粒度へ計測点が拡張されている
- `/usage` と Rate Limits API の並行整備は、利用者UIと管理者APIの両方で制限状態を可視化する設計を示す
- AIエージェントの運用では、失敗を減らすことと同じくらい、失敗や制限を説明可能にすることが重視されている

Claude Codeの更新は、Anthropic純正環境に閉じていない。Bedrock、Vertex AI、Windows、PowerShell、WSL2、devcontainers、tmux、VSCodeなど、異種環境での動作修正が継続的に入っている。

- WindowsでBash前提を外し、PowerShellを主シェルとして扱う変更は、利用者に環境適応を求めるのではなく、製品側が実環境に寄せる方向を示している
