# iPhoneからの指示でMac上の作業を完了するAnthropic「Claude」のComputer UseとDispatchの実像

## Executive Summary

Anthropicは、スマートフォンからClaudeにタスクを送ると、ユーザーのコンピューター上でClaudeがアプリ起動、ブラウザ操作、スプレッドシート入力まで自律実行する「Computer Use」を試験提供している。単なるチャット応答ではなく、Dispatchによる継続対話と組み合わせて、端末上のファイルやアプリに触れる実務エージェントへ踏み込んだ点が焦点である。

同社はこの機能をOpenClawに対抗するAIエージェント戦略の一環として位置づける一方、「Claude can make mistakes」と明言し、新しいアプリへのアクセス時には必ず許可を求める設計を採っている。自律性の拡大と安全策の明示が、同時に進んでいる。

## Claudeの「Computer Use」：スマホ起点でPC上の操作を完遂する新機能

Anthropicは、ユーザーが「phone」からClaudeにタスクを送ると、Claudeがその人の「computer」で作業を完了する機能を発表した。記事では、Claudeがユーザーのコンピューターを使ってタスクを完了できるようになったと説明されている。

- **エビデンス**
- CNBC記事のKey Pointsは「Anthropic is trialling a feature that lets users send prompts to Claude from a smartphone.」と記載。
- 同じくKey Pointsで「Claude will complete the task on its own on a person's computer.」と記載。
- 本文でAnthropicは「Users can now message Claude a task from a phone, and the AI agent will then complete that task」と説明。
- Claudeは「open apps on your computer, navigate a web browser and fill in spreadsheets」とされている。
- **結論**
- 機能の中心は、自然言語での支援ではなく、PC上のGUI操作そのものをClaudeが代行する点にある。
- 入力インターフェースはスマホ、実行環境はユーザーPCという分離により、外出中や移動中の指示を既存業務端末上で処理させる利用形態が前提になっている。
## 実演で示された作業範囲：PDF書き出しと会議招待への添付

Anthropicは動画で、会議に遅れそうなユーザーがClaudeに資料送付を依頼するユースケースを示した。ここでのポイントは、単一アプリ内の応答ではなく、ファイル操作と会議関連アクションをまたぐ一連の実務処理を実行していることである。

- **エビデンス**
- 記事は「One prompt Anthropic demonstrated in a video posted Monday is a user running late for a meeting.」と記載。
- ユーザーは「export a pitch deck as a PDF file and attach it to a meeting invite」と依頼。
- 記事は「The video shows Claude carrying out the task.」と述べている。
- **結論**
- ClaudeのComputer Useは、文書生成や要約ではなく、既存ファイルを扱う事務処理の自動化に踏み込んでいる。
- 「export」「attach」という操作が示すのは、アプリ間をまたぐワークフロー実行であり、従来のチャットボットよりもOS操作に近い役割である。
## OpenClaw対抗としてのAIエージェント競争

CNBCは、Anthropicの今回の発表を「agents」競争の文脈で位置づけている。特に2026年に話題化したOpenClawが基準点として扱われ、Claudeの新機能はそれに対抗する動きとして描かれている。

- **エビデンス**
- 記事はAnthropicが「looks to create an AI agent that can rival the viral OpenClaw」と記載。
- 「Agentic capabilities were thrust into the spotlight this year after the release of OpenClaw, which went viral.」
- OpenClawについて「links to AI models from OpenAI and Anthropic」と説明。
- さらに「A user can message OpenClaw through popular apps like WhatsApp or Telegram to carry out tasks.」
- Anthropicの新機能と同様に、OpenClawは「runs locally on a user's device giving it access to files」とされる。
- Nvidia CEO Jensen HuangはCNBCに対し、OpenClawは「definitely the next ChatGPT」と述べた。
- Nvidiaは「last week announced NemoClaw, an enterprise-grade version of OpenClaw.」
- OpenAIは「last month hired Peter Steinberger, the creator of OpenClaw」とされる。
- **結論**
- ClaudeのComputer Useは単独の製品改善ではなく、OpenClawが可視化した「個人向けエージェント」の競争軸に対する応答である。
- ローカル端末上で動き、ファイルへアクセスできることが、2026年時点のエージェント差別化要件として浮上している。
- Nvidia、OpenAI、Anthropicが同じ競争領域に集まりつつあり、評価対象はモデル性能だけでなく、実際に仕事を終わらせる実行能力へ移っている。
## Dispatchと安全策：継続対話と権限制御の組み合わせ

Anthropicは、Computer Useを単発操作ではなく、Dispatchを通じた継続的なタスク委任の文脈で提供している。同時に、機能の未成熟さとリスクを明言している。

- **エビデンス**
- 記事は「Users can use Dispatch, a feature it released last week in Claude Cowork.」と記載。
- Dispatchは「lets users have a continuous conversation with Claude from a phone or desktop and assign the agent tasks.」
- Anthropicはcomputer useについて「is still early compared to Claude's ability to code or interact with text.」と警告。
- さらに「Claude can make mistakes, and while we continue to improve our safeguards, threats are constantly evolving」と述べている。
- 同社は「with safeguards that minimize risk」とし、「Claude will always request permission before accessing new apps.」と説明している。
- **結論**
- Dispatchは、単発の命令実行ではなく、スマホ/デスクトップをまたいだ継続対話型の業務委任レイヤーとして機能する。
- 一方でAnthropic自身が、コード生成やテキスト対話よりComputer Useの成熟度が低いと位置づけているため、完全無監督の信頼性を前提とした運用には慎重さが必要である。
- 「new apps」へのアクセス時に都度許可を求める設計は、OS全域への無制限な自律操作ではなく、権限境界を残したエージェントとして実装していることを示す。
